# Informe del Facilitador-Administrador

  Se considera que este rol facilita la configuración y el
  funcionamiento del grupo. La persona encargada de desempeñar este
  rol actúa como líder del grupo y es responsable de la distribución
  de tareas, la mediación de conflictos, la comprobación del
  cumplimiento de las obligaciones y la motivación y ánimo de sus
  compañeros. Es no solo el encargado de que la tarea se haga, sino de
  que la salud mental del grupo se mantenga en niveles deseables.
  
  Además, se encarga de revisar en el repositorio de _github_ el
  trabajo del grupo.


## Semana 1

1. Diseño de la Interfaz de la Aplicación:

Hemos completado el proceso de diseño de la interfaz de usuario de la aplicación, centrándonos específicamente en el dispositivo móvil. 
El diseño sigue las normas, pautas establecidas, y casos de usos marcados abordando los siguientes puntos:

La interfaz ha sido diseñada como un único documento o página web, optimizado para dispositivos móviles.
Se han integrado elementos  para la gestión de errores.


2. Implementar el diseño:

Hemos creado la interfaz de usuario para nuestra aplicación utilizando html, css y javascript, siguiendo las reglas del W3C. 
En resumen, organizamos el contenido de manera lógica con HTML semántico.


## Semana 2

1. Mejora del Diseño Responsivo:

Hemos extendido el diseño de la interfaz para garantizar una experiencia de usuario óptima en diversos dispositivos. Nos centramos en los casos base típicos, 
como smartphone, tablet y desktop. La interfaz ahora se adapta de manera gradual según el tamaño de la pantalla y la configuración del navegador.


2. Modificación de la Implementación:

Para lograr una interfaz adaptable, realizamos modificaciones en el styles.css.


3. Compatibilidad Cross-Browser:


Hemos llevado a cabo pruebas exhaustivas en varios navegadores populares.
Esto asegura una experiencia consistente para todos los usuarios, independientemente del navegador que utilicen.


## Semana 3

En la tercera semana de desarrollo, nos enfocamos en mejorar la accesibilidad de la aplicación web mediante la implementación de 
WAI-ARIA (Web Accessibility Initiative - Accessible Rich Internet Applications).

1. Integración de WAI-ARIA:
Hemos modificado la implementación existente para cumplir con las normas de accesibilidad en aplicaciones web WAI-ARIA. Para ello, 
hemos identificado las necesidades de la implementación considerando la lista de casos de uso, y aplicamos las reglas pertinentes del WAI-ARIA en cada caso.
