# Informe del Curador-Traductor

  El traductor debe definir -con sus propias palabras- los objetivos
  de aprendizaje de la práctica.

  El curador buscará los recursos online que ayuden al equipo a
  completar de mejor manera su tarea.
  
  Además, el curador es el responsable de recopilar y organizar de
  forma esquemática, todas las fuentes adicionales de información que
  el grupo ha utilizado para desarrollar la actividad.

  Finalmente, el curador tiene la tarea de presentar y explicar a sus
  compañeros los recursos indentificados.

  > A continuación se ofrece una sugerencia sobre el contenido del
  > informe que debe generar la persona encargada de este role cada
  > semana.



# OBJETIVOS DE APRENDIZAJE GENERALES 

- Diseño de Interfaces Gráficas: desarrollar habilidades en el diseño de interfaces gráficas adaptativas para aplicaciones móviles.

- Patrones Arquitectónicos y Estado de la Aplicación: aplicar patrones arquitectónicos para gestionar el estado de la aplicación de manera eficiente.

- Implementación de Interfaces Gráficas con Flutter:  implementar la aplicación diseñada utilizando Flutter, haciendo uso de librerías para construir 
interfaces gráficas atractivas.

- Desarrollo de Pruebas End-to-End: implementar tests para casos de uso de la aplicación, cubrir situaciones de errores de E/S y errores del usuario.

- Flutter y Dart: usar Dart y Flutter para construir una aplicación funcional y eficiente.



# OBJETIVOS ESPECÍFICOS PARA CADA TAREA 


TAREA 1: Diseño de la interfaz:

- Identificación y Documentación de Casos de Uso.
- Diseño de Interfaz de Usuario (IU).
- Selección de Patrón Arquitectónico (Provider Model)
- Diseño Software Siguiendo el Patrón Seleccionado


TAREA 2: Implementación de la aplicación:

- Implementación de la Aplicación en Flutter
- Gestión de Errores en Entrada/Salida (E/S)
- Pruebas en Dispositivos Móviles
- Programación Dirigida por Eventos
- Uso de Concurrencia con Funciones Asíncronas


TAREA 3: Test end2end:

- Implementación de Tests End-to-End
